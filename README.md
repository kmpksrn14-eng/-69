# 📊 ระบบ Dashboard ติดตามโครงการสำหรับผู้บริหาร โรงเรียนวัดคูยาง
(Wat Khuyang School Executive Dashboard)

ระบบแสดงผลและติดตามโครงการ/กิจกรรมเชิงยุทธศาสตร์และงบประมาณ สำหรับผู้บริหารและคณะครูโรงเรียนวัดคูยาง ข้อมูลทั้งหมดจะดึงจาก Google Sheets โดยตรงและสรุปผลอัปเดตใหม่แบบเรียลไทม์ทุกครั้งเมื่อเปิดเว็บหรือกดรีเฟรช

---

## ✨ คุณลักษณะของระบบ (Features)
- 📈 **การเชื่อมโยงข้อมูลแบบเรียลไทม์**: โหลดข้อมูลจาก Google Sheets เสมอเมื่อเปิดหน้าเว็บ (โหลดข้อมูลสรุปใหม่ทันทีเมื่อมีการอัปเดตฝั่งชีต)
- 💼 **สรุปผลสถิติสำหรับผู้บริหาร (Executive KPIs)**:
  - จำนวนโครงการทั้งหมด, งบประมาณที่ตั้งไว้ทั้งหมด
  - งบประมาณที่ใช้ไปจริง (สัดส่วนร้อยละ), งบประมาณคงเหลือ
  - สรุปแยกสถานะโครงการ (ดำเนินการแล้ว, อยู่ระหว่างดำเนินการ, ยังไม่ดำเนินการ)
  - คำนวณความคืบหน้าเฉลี่ยของแผนงานทั้งหมด
- 📊 **กราฟสรุปข้อมูลแบบอินเตอร์แอคทีฟ (Interactive Charts)**:
  - กราฟแท่งเปรียบเทียบงบประมาณจัดสรร vs เบิกจ่ายจริง รายโครงการ
  - กราฟวงกลม (Doughnut Chart) แสดงสัดส่วนสถานะการดำเนินการ
  - กราฟแท่งแสดงงบประมาณรวมจำแนกตามกลุ่มงาน/ฝ่าย
- 🔍 **ระบบค้นหาและกรองข้อมูลขั้นสูง**:
  - ค้นหาด้วยชื่อโครงการ หรือชื่อผู้รับผิดชอบได้แบบพิมพ์ไปค้นหาไป
  - ตัวกรองแยกตาม "กลุ่มงาน/ฝ่าย" (สร้างตัวเลือกโดยอัตโนมัติตามข้อมูลจริง)
  - ตัวกรองแยกตาม "สถานะการดำเนินงาน"
- 🌓 **ระบบธีมสีสว่าง / สีเข้ม (Light & Dark Theme)**: เลือกปรับระดับความสบายสายตา พร้อมจำตัวเลือกไว้ใช้งานในครั้งถัดไป
- 📱 **การออกแบบที่ยืดหยุ่น (Responsive Design)**: รองรับหน้าจอคอมพิวเตอร์ แท็บเล็ต และสมาร์ทโฟน

```text
MM/

├── index.css    # การตกแต่งสไตล์ หรูหราแบบ Glassmorphism และระบบธีม

└── README.md    # คู่มือแนะนำการติดตั้งและการเผยแพร่
```

## 📊 แหล่งข้อมูล (Google Sheets)
ลิงก์แชร์ของ Google Sheets หลัก:  
[ลิงก์ Google Sheets สำหรับแก้ไขข้อมูล](https://docs.google.com/spreadsheets/d/1-J7D5APDRZOTz_X1cqOCvJb3KHdZ4QQcfAJGbrHCQh4/edit?usp=sharing)

**เงื่อนไขสำคัญเพื่อให้ Dashboard โหลดข้อมูลได้:**
1. หน้า Google Sheets ดังกล่าวจะต้องแชร์เป็นสาธารณะ (ผู้ที่มีลิงก์สามารถดูได้)
2. โครงสร้างคอลัมน์มาตรฐาน:
   - คอลัมน์ที่ 1: ชื่อโครงการ/กิจกรรม
   - คอลัมน์ที่ 2: ผู้รับผิดชอบ
   - คอลัมน์ที่ 3: กลุ่มงาน/ฝ่าย
   - คอลัมน์ที่ 4: งบประมาณที่ตั้ง
   - คอลัมน์ที่ 5: งบประมาณที่ใช้ไป
   - คอลัมน์ที่ 6: งบประมาณคงเหลือ (ระบบสามารถคำนวณแทนได้หากไม่ได้กรอก)
   - คอลัมน์ที่ 7: ความคืบหน้า (%)
   - คอลัมน์ที่ 8: สถานะ (`ยังไม่ดำเนินการ`, `อยู่ระหว่างดำเนินการ`, `ดำเนินการแล้ว`)

## 🚀 วิธีการนำขึ้น GitHub เพื่อเผยแพร่ (How to Deploy to GitHub)

เนื่องจากระบบนี้เป็นแบบ **Client-Side Static App** (ไม่ใช้ฐานข้อมูลฝั่งหลังบ้าน) คุณสามารถโฮสต์เว็บนี้ได้ **ฟรี 100%** ผ่านบริการ **GitHub Pages** ตามขั้นตอนง่ายๆ ดังนี้ครับ:

### วิธีที่ 1: อัปโหลดผ่านหน้าเว็บไซต์ GitHub (ไม่ต้องติดตั้งโปรแกรมใดๆ)
1. เข้าไปที่เว็บ [GitHub](https://github.com/) และลงชื่อเข้าใช้งาน (สมัครฟรี)
2. คลิกปุ่ม **"New"** (หรือ **"Create repository"**) เพื่อสร้างพื้นที่เก็บข้อมูลใหม่
3. ตั้งชื่อ Repository เช่น `watkhuyang-dashboard`
4. ตั้งค่าระดับการเข้าถึงเป็น **Public** (จำเป็นสำหรับใช้งาน GitHub Pages ฟรี)
5. คลิกปุ่ม **"Create repository"** ด้านล่างสุด
6. ในหน้าถัดมา ให้มองหาข้อความ **"uploading an existing file"** แล้วคลิกที่ลิงก์นั้น
7. ลากไฟล์ในเครื่องคอมพิวเตอร์ของคุณ ทั้ง 3 ไฟล์ (`index.html`, `index.css`, `app.js`) ไปวางในช่องอัปโหลดหน้าเว็บ
8. กดปุ่ม **"Commit changes"** สีเขียวด้านล่าง
9. **ขั้นตอนเปิดระบบเว็บบล็อก:**
   - ไปที่แท็บ **Settings** ด้านบนของหน้า Repository
   - เมนูด้านซ้ายมองหาหัวข้อ **Pages**
   - ในหัวข้อ **Build and deployment** > **Branch** ให้เปลี่ยนจาก `None` เป็น `main` (หรือ `master`)
   - กดปุ่ม **Save**
10. รอประมาณ 1-2 นาที GitHub จะแสดงลิงก์ที่เข้าถึงแดชบอร์ดได้ เช่น `https://[ชื่อผู้ใช้ของคุณ].github.io/watkhuyang-dashboard/` คุณสามารถส่งลิงก์นี้ให้ผู้บริหารตรวจสอบได้ทันที!

---

### วิธีที่ 2: อัปโหลดผ่าน Git Command Line
หากคุณมี Git ติดตั้งในเครื่องคอมพิวเตอร์แล้ว สามารถรันชุดคำสั่งด้านล่างนี้ได้เลยครับ:

1. เปิด PowerShell หรือ Command Prompt ที่โฟลเดอร์โครงการนี้:
   ```bash
   # สร้าง Git Repository ในเครื่อง
   git init
   
   # แอดไฟล์ทั้งหมดเข้าระบบติดตาม
   git add .
   
   # บันทึกไฟล์เวอร์ชันแรก
   git commit -m "Initial Wat Khuyang dashboard deployment"
   
   # สร้างชื่อกิ่งหลัก
   git branch -M main
   
   # เชื่อมโยงกับ GitHub Repository ปลายทาง (แก้ไขลิ้งค์เป็นของคุณเอง)
   git remote add origin https://github.com/[ชื่อผู้ใช้ของคุณ]/[ชื่อคลังข้อมูล].git
   
   # อัปโหลดไฟล์ขึ้น GitHub
   git push -u origin main
   ```
2. ทำตามขั้นตอนข้อ 9-10 ในวิธีที่ 1 ด้านบน เพื่อเปิดฟังก์ชัน **GitHub Pages**

## 🛠️ รายละเอียดทางเทคนิคของระบบ
- **CSS Variable / Light-Dark Themes**: การเปลี่ยนธีมสลับสไตล์แบบนุ่มนวลโดยเพิ่มคลาส `.dark-theme` หรือ `.light-theme` เข้าไปที่ส่วน `body`
- **Cache-Busting Fetch**: มีการต่อท้ายคิวรีสติงตัวแปลเวลาปัจจุบัน `?_cb=[Timestamp]` เข้าไปท้าย URL ดึงข้อมูล เพื่อป้องกันเบราว์เซอร์แสดงผลหน้าเพจเก่าที่แคชไว้ ช่วยให้ดึงค่าชีตล่าสุดชัวร์ 100% ทุกครั้งที่เปิดเว็บ
- **Defensive CSV Parsing**: ระบบมีโค้ดตรวจสอบแถวหัวตารางอัตโนมัติ หากแถวแรกของ Google Sheet มีข้อความประเภท "งบประมาณ" หรือไม่ใช่ตัวเลข ระบบจะข้ามไปประมวลผลแถวถัดไป ทำให้ยืดหยุ่นต่อกรณีที่คุณมีหัวตาราง
- **Chart.js & PapaParse**: โหลดผ่าน CDN ที่มีความเร็วการเข้าถึงสูงและเสถียร (เสร็จสิ้นในหน้าเดียว ไม่กินทรัพยากรเครื่อง)
